# Salesdata_Analysis

Project Overview
This project analyzes a sales dataset using Python to gain insights into sales performance across regions, products, and salespersons. The analysis includes data cleaning, preprocessing, exploratory data analysis (EDA), and data visualization to support business decision-making.

Objectives
- Clean and prepare raw sales data for analysis.
- Identify sales trends and patterns.
- Compare sales performance across different regions.
- Evaluate product-wise sales performance.
- Analyze salesperson contributions.
- Study monthly sales trends.

Dataset Description
The dataset contains sales transaction records with the following attributes:

| Column | Description |
|----------|-------------|
| OrderID | Unique order identifier |
| OrderDate | Date of the order |
| Region | Sales region |
| Product | Product sold |
| SalesPerson | Name of salesperson |
| Quantity | Number of units sold |
| UnitPrice | Price per unit |
| TotalAmount | Total sales amount |

Data Cleaning and Preprocessing
Several data preparation steps were performed to improve data quality:

- Checked for missing values.
- Filled missing values using appropriate methods:
  - Mode for categorical columns.
  - Mean for numerical columns.
- Corrected invalid data entries.
- Converted date columns into datetime format.
- Removed duplicate records.
- Renamed the `TotalAmount` column to `Sales` for better readability.
- Verified data types and consistency.

Exploratory Data Analysis (EDA)

1. Regional Sales Analysis
- Calculated total sales generated by each region.
- Identified high-performing and low-performing regions.
- Visualized results using bar charts.

2. Product Performance Analysis
- Measured total sales for each product.
- Compared product contributions to overall revenue.
- Identified best-selling products.

3. Salesperson Performance
- Analyzed quantity sold by each salesperson.
- Compared individual performance levels.

4. Monthly Sales Trend Analysis
- Extracted month information from order dates.
- Evaluated monthly sales fluctuations.
- Identified peak sales periods.

5. Pricing Analysis
- Calculated average unit price for each product.
- Compared pricing patterns across products.

Technologies Used

- Python
- Pandas
- Matplotlib
- Seaborn
- Jupyter Notebook

Visualizations
The project includes various visualizations such as:

- Bar Charts
- Histograms
- Pie Charts / Donut Charts
- Monthly Trend Charts

These visualizations help present sales insights in a clear and understandable manner.

Key Insights
- Regional performance differences were identified.
- Certain products contributed significantly more revenue.
- Salesperson productivity varied across the dataset.
- Monthly trends highlighted periods of strong and weak sales activity.
- Pricing analysis provided insight into product value and revenue generation.

Project Files

- `sales_dataset.csv` – Raw sales dataset.
- `SalesAnalysis.ipynb` – Jupyter Notebook containing data cleaning, analysis, and visualizations.

How to Run the Project

1. Clone or download the project files.
2. Install the required libraries:

```bash
pip install pandas matplotlib seaborn
```

3. Open Jupyter Notebook:

```bash
jupyter notebook
```

4. Open `SalesAnalysis.ipynb`.
5. Run all cells to reproduce the analysis and visualizations.

Conclusion
This project demonstrates the complete workflow of sales data analysis, including data cleaning, exploration, visualization, and insight generation. It serves as a beginner-friendly example of how Python can be used to transform raw sales data into meaningful business insights.
